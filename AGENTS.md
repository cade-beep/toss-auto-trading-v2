## Overview

This repository is a Next.js + Supabase AI trading workstation for Toss-style automated trading. Treat trading changes as high risk: settlement correctness, auditability, and recovery matter more than speed.

Current product direction:

- Simulation mode now
- Future Paper Trading mode
- Future Live Toss Open API mode
- Portfolio, positions, watchlist, and order persistence
- AI/rule-based strategy execution with explicit risk controls

## Supabase And Execution Rules

Current tables:

- `portfolio`
- `positions`
- `watchlist`
- `orders_log`

RLS is enabled on all current public tables. Do not add exposed public tables without RLS policies.

`execute_trade` is the authoritative settlement path for filled executions. It reads `auth.uid()`, locks the user's `portfolio`, mutates cash/positions, and inserts into `orders_log`.

Do not bypass `execute_trade` for portfolio or position settlement. Future Paper/Live fills should create durable execution records and call `execute_trade` once per accepted fill with an idempotent execution id.

## Trading Safety Requirements

Never assume an order succeeded. Any broker-like flow needs:

- persistent order state
- idempotency via `client_order_id` or equivalent
- partial-fill handling
- cancellation/replacement handling
- reconciliation after network or broker failures
- audit logs for decisions, state changes, and fill settlement

Future architecture should preserve:

- PaperTradingService: real market data, simulated fills, no real broker order submission
- Risk Engine between Strategy Execution and `TradingService`
- MarketDataProvider as the canonical quote source
- Strategy Worker as a server-side producer of order intents
- Broker Order Mapping for client ids, broker ids, fills, cancels, replacements, and reconciliation
- Future Toss Open API integration through `TossTradingService` only

## Frontend Guidance

This is a trading workstation, not a marketing site.

Prefer compact operational panels, clear order/position state, restrained color use, accessible contrast, predictable controls, and fast manual trading workflows.

Avoid generic SaaS landing-page composition, decorative gradients/orbs, oversized hero sections, and visuals that obscure trading state.
